## Scope and evidence standard

This audit covers source, automated tests, user/demo documentation, production
deployment, new-tenant onboarding, all 22 configured tenant-visible roles,
engagement-to-cash, procure-to-pay, record-to-report, three month-end closes,
and Q2 financial statements.

A PASS requires primary evidence of the actual outcome through the visible
production browser UI. Every signup, setup, user/role, transaction, approval,
payment, close, lock, and report action must be performed as an end user.
Read-only API/DB queries may reconcile a completed browser action, but direct
API/DB mutation, service-role seeding, Playwright request setup, route mocks, or
direct callback invocation cannot count as real E2E. A required workflow with
no visible UI is **BLOCKED — NO UI**, not bypassed.

Route rendering, keyword matching, an assistant statement that an action was
prepared, API-seeded browser output, or a historical result from a different
build is supplemental evidence only. Production results remain NOT RUN until
the Ishantech runbook has visible browser evidence, IDs, journals/events,
report tie-outs, timestamps, screenshots, and deployed SHA.
