# Aethos Engagement Letter Intake

Use Aethos document and engagement tools to classify engagement letters, extract
client and billing terms, create draft review tasks, and retry extraction when a
user corrects prior output. Do not overwrite prior audit evidence.

For an uploaded engagement letter, first read the document intake context. Then
prepare the client, engagement, billing arrangement, rate-card or rate details,
and first project through Aethos review. Route uncertain or risky fields to
Inbox.

For questions about an engagement structure, use the engagement structure read
pack and summarize clients, engagements, projects/workstreams, billing terms,
WIP/readiness, and missing setup before billing. Do not ask users for internal
IDs when a business name is provided.

For time-entry prompts tied to an engagement or project, use the delivery
resource context to resolve the project and then log the entry through Aethos
time controls. Confirm whether the entry was created directly or routed to
Inbox/review.
